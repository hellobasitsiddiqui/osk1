# android — Capacitor Android shell (OSK-146)

A thin native Android host that wraps the existing `web/` app in a WebView using
[Capacitor](https://capacitorjs.com/). This is the root of the MOBILE epic.

**Mobile is a hybrid over the web codebase, not a native rewrite.** The `web/`
static app (index / app / status / ops / help `.html` + `config.js` / `auth.js`
+ the feature modules) *is* the application. Capacitor only provides the native
shell (an Android WebView) so that push, GPS, camera and biometrics can be added
later as Capacitor plugins — each its own ticket — while `web/` keeps shipping to
Firebase Hosting on its existing instant-deploy cadence. The web build/deploy is
completely unchanged; this `android/` project is **additive**.

## How it fits together

| Piece | Where | Committed? |
| --- | --- | --- |
| Capacitor tooling manifest | `../package.json` (repo root) | yes |
| Capacitor config (`appId`, `appName`, `webDir: web`) | `../capacitor.config.ts` | yes |
| Generated native Android project | this `android/` dir | yes (source only) |
| Copy of `web/` served by the WebView | `android/app/src/main/assets/public` | **no** — git-ignored, regenerated by `cap sync` |

`npx cap sync android` copies `web/` into `android/app/src/main/assets/public`
and refreshes native dependencies. That copy, the generated
`capacitor.config.json` / `capacitor.plugins.json`, and
`capacitor-cordova-android-plugins/` are all git-ignored (see `android/.gitignore`)
and regenerated on demand — so the committed tree is source only, never a stale
duplicate of `web/`.

Key coordinates (from `../capacitor.config.ts` and the generated project):

- **appId / applicationId:** `dev.openskeleton.app`
- **appName:** `OpenSkeleton`
- **minSdk:** 24 &nbsp;·&nbsp; **compileSdk / targetSdk:** 36 (see `android/variables.gradle`)
- **Gradle wrapper:** 8.14.3 &nbsp;·&nbsp; **Capacitor:** 8.4.1

## Build / run locally

Prerequisites: **Node ≥ 22** (the Capacitor 8 CLI aborts on Node 20), **JDK 21**
(Capacitor 8's `capacitor-android` library compiles with `--release 21`, so
JDK 17 fails), and the **Android SDK** (platform 36 + build-tools 35.0.0),
easiest via [Android Studio](https://developer.android.com/studio).

```bash
# 1. from the repo root: install the Capacitor tooling
npm ci

# 2. copy the current web/ app into the native project + refresh native deps
npx cap sync android

# 3a. open the project in Android Studio (Run ▶ on a device/emulator)
npx cap open android

# 3b. …or build a debug APK straight from the CLI
cd android && ./gradlew assembleDebug
#   output: android/app/build/outputs/apk/debug/app-debug.apk
```

`./gradlew` needs `ANDROID_HOME` (or `ANDROID_SDK_ROOT`) pointing at your SDK, or
an `android/local.properties` with `sdk.dir=…` (Android Studio writes this
automatically; it is git-ignored and must never be committed).

Re-run `npx cap sync android` after **every** change to `web/` — otherwise the
APK keeps serving the previously copied assets.

## Verified in CI

`.github/workflows/android.yml` is **path-gated** — it runs only when the mobile
scaffold changes (`android/**`, `capacitor.config.ts`, `package.json`,
`package-lock.json`, the workflow itself) so unrelated PRs never pay for the slow
Android runner. It installs JDK 21 + Node 22 + the Android SDK, then runs
`npm ci` → `npx cap sync android` → `./gradlew assembleDebug` as a compile check,
and uploads the resulting unsigned debug APK as a run artifact. No signing
happens.

Note `web/**` is intentionally **not** a trigger: `cap sync` only copies the
static web assets into the APK, so a web content change cannot break the native
`assembleDebug` compile — gating the slow Android job on the frequently edited
`web/` tree would run it on nearly every PR for no benefit.

## Deliberately deferred (later MOBILE-epic tickets)

This ticket is the **scaffold only**. Out of scope here, tracked separately:

- **Native plugins** — push notifications, GPS/geolocation, camera, biometric
  auth (added as Capacitor plugins + the shared `webview/` bridge).
- **Release signing + distribution** — keystore, signing config, Play Store /
  internal-track upload (OSK-128 / OSK-153 / OSK-157 / OSK-159). No keystore or
  secret lives in this repo; that is a human-owned ticket.
- **App theming** — splash screen, status-bar styling, adaptive launcher icon.
- **Deep links / universal links** back into the web routes.
- **On-device / instrumented e2e** tests.
- **iOS shell** — a sibling `ios/` project in a later epic (the `webview/` bridge
  is authored once both shells exist).
